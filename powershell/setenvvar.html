Windows+PowerShell: Set Environment Variable by PowerShell

Introduction

Setting the environment var in the middle of a configuration or when writing a script is always a frustration, especially when you are not sure where to look for, and what to look for...Is it System, Local, environment.

A PowerShell script saves lot of reserach energy and time if you have a handy one ready with you.

Categories

  • 1. Readiness probes
  • 2. Liveness probes

If you think that setting an env var is easy for you, please skip this article , as it can be achieved too easy in many other ways.

I really considered the compliance of checking and setting a variable at the expense of a simple check and execute commands.

Here we go the shelly way....

						
	$oldPath = [Environment]::GetEnvironmentVariable('Path', [EnvironmentVariableTarget]::Machine)
	if ($oldPath.Split(';') -inotcontains 'C:\minikube'){ `
		[Environment]::SetEnvironmentVariable('Path', $('{0};C:\minikube' -f $oldPath), [EnvironmentVariableTarget]::Machine) `
	}

The command does the following things:
  • Grabs the Machine Environment Variable Path
  • Checks if the New Path exists
  • If not it adds it to the Environment Variables
