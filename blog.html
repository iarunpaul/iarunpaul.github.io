Grafana with Azure AD Authentication

In this article, we’re going to know about, How we can do authentication in Grafana with AAD?

Note: This article is continue of my previous article Deploy Grafana to Azure App Service. So, please go through with that article before proceeding further.

Pre-requisite

The following are the pre-requirements:

  • First, go through my previous article for better understanding and, complete all steps.
  • You should have enough access to the Azure AD as well as in App registrations to do some opeartions,
  • later, download the Grafana for windows as zip file, and extract it.

Now, if you did everything as mentioned above, then we’re good to go.

Well, doing this is pretty much easy, only you just have to follow the exact process which we’re going to see here.

So, the following things which we’re going to cover are as follows:

  1. Create and configure an App in Azure portal.
  2. Assign Users or, Groups to an App in AAD.
  3. Create a Grafana custom config file.
  4. Upload the Grafana config file to Azure File shares.

So, let’s get started.

Create an App in Azure Portal

To create an App, just do the step-by-step process:

  • Search App registrations in Azure portal, then click on + New registrations button,
  • then, fill the Name of the App, and click on the Register button,
  • Now, copy the Client ID, Client Secret & Tenant ID present on the Overview section of an App, and save it to any editor for later use.

To generate the Client secret, do the following steps:

  1. Navigate to Certificates & secrets,
  2. then, click on + New client secret,
  3. and, fill the Description and Expires (select months as per your requirement) field,
  4. then, click on Add button.
  5. lastly, copy the Value (secret) and save it somewhere for later use.

Note: Copy the Secret value as soon as secret generated because the Value will not be visible completely once you leave the section (Certificates & secrets).

Define Roles to an App

  • Now, will define the roles (Admin, Editor & Viewer) to an App. To do so, copy the below code and paste it to the “appRoles”:[] section in Manifest of App. If not defined, all users will have the Viewer role.

Note: The roles in App can be added in two ways, one is from App roles → + Create app role, another is from Manifest which we have mentioned here.

"appRoles": [     {        "allowedMemberTypes": [             "User"       ],       "description": "Read only Users",       "displayName": "Viewer",       "id": "SOME_UNIQUE_ID",       "isEnabled": true,       "lang": null,       "origin": "Application",       "value": "Viewer"     },     {       "allowedMemberTypes": [            "User"       ],       "description": "Admin Users",       "displayName": "Admin",       "id": "SOME_UNIQUE_ID",       "isEnabled": true,       "lang": null,       "origin": "Application",       "value": "Admin"     },     {       "allowedMemberTypes": [            "User"       ],       "description": "Editor Users",       "displayName": "Editor",       "id": "SOME_UNIQUE_ID",       "isEnabled": true,       "lang": null,       "origin": "Application",       "value": "Editor"     }],

Note: Every role requires a unique ID (id) which you can generate on Linux with uuidgen, and on Windows through PowerShell with New-Guid.

  • You can checkout the added roles in App under App roles.

Add Redirect URLs to an App

To add the redirect URLs, do the following steps:

  1. Navigate to the Authentication,
  2. Add the Web App URL under Web platform, then click on Save button.
https://<domain-name>/login/azuread

Note: You can add Redirect URLs, as much as you need to an App.

Assign Users/Groups to an App in Azure AD

Once you did all the above steps, let’s go to Azure AD to assign Users/Groups to an App. To do this, follow the belows steps:

  • Navigate to Azure Active Directory,
  • then, go to the Enterprise applications, and search your App (In this case — Grafana OAuth2 Login) under All applications, and then click on it,
  • After this, navigate to Users and groups, and click on + Add user/group,
  • Now, click on None Selected under Users to add an user/group and then, click on None Selected under Select a role to assign a role to an user/group.
    Lastly, click on Assign button.
  • Note the added Groups ID (object Id) to add it later into Grafana config file. You can get this, by navigating to Azure AD → Groups.

Note: The Groups Id MUST be only copied/noted, when you have assigned any Groups to an App. Otherwise, NO need to note it.

Create a Grafana Custom Config File

  • As per the Grafana, the default settings for a Grafana instance are stored in the $WORKING_DIR/conf/defaults.ini file. Grafana defaults are stored in this file.

Note: Do not change the location in this file as well as Do not change defaults.ini!

  • So, to create custom.ini or grafana.ini file, you just need to Copy sample.ini and name it custom.ini . sample.ini is in the same directory as defaults.ini and contains all the settings commented out.

Note: These settings are for only Windows. For other OS, and more details about this, checkout the link.

  • Now, add the following to the custom.ini :
[auth.azuread]
name = Azure AD
enabled = true           #Default is false
allow_sign_up = true
client_id = APPLICATION_ID
client_secret = CLIENT_SECRET
scopes = openid email profile
auth_url = https://login.microsoftonline.com/TENANT_ID/oauth2/v2.0/authorize
token_url = https://login.microsoftonline.com/TENANT_ID/oauth2/v2.0/token
allowed_domains =
allowed_groups =
  • To make it work properly, add the follwoing as well and save it:
[server]
domain = <ADD-DOMAIN-NAME>
root_url = <ADD-FULL-URL>

These thing, you can find out as shown in the image:

So, in our case, the domain and root_url can be grafana-blog.azurewebsites.net and https://grafana-blog.azurewebsites.net respectively.

Note: If you want to add allowed_domains and allowing_groups, checkout the configuraton here.

Upload Custom Config File to Azure File Share

Now, will upload the configured custom.ini file to the Azure File Share.
To do so, do the following steps:

  • Navigate to the portal
  • Under Resource group, click on Storage account,
  • then, navigate to File shares, and click on the exisitng File share,
  • at last, click on ⤒ Upload button and select the custom.ini file and upload it.

Note: Directories — csv, plugins, png and files — grafana.db-shm, grafana.db-wal which you are seeing in the above image are auto generated and you no need to upload it seperately.

Add Mount path for Config file in App Service

To make the authentication work completely, need to add mount path for config file in Web App.

To make this, do the following:

  • Navigate to Web App,
  • then, click on Configuration and navigate to Path mappings,
  • then, click on + New Azure Storage Mount, and fill the following config and save it.
  • lastly, click on 💾 Save button.
  • Now, navigate to Overview, click on Refresh button. At last, click on URL and, you can see the login screen with Sign in with Microsoft button.

Note: It will take some time to load the Grafana Login screen as it’s loading all the configuration from config file.

Conclusion

Well, you just enabled the Azure AD Authentication for Grafana.

Likewise, you can also do for Google, Github, etc., with some little change in configuration.
