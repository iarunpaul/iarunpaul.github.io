Deploy Grafana to Azure App Service using Bicep

Well, as per my previous article, which says, we can deploy Grafana to Azure App Service by creating each resources manually.

But, the challenge will come when we have to do the same thing for different environment(Dev, Test, UAT, Prod). It’s difficult to manage and maintain each resources for each environments as it increases the more manual intervantion and the rate of mistakes.

So, to mitigate this issue, we’re going to use Bicep. There is some reason why I’ve choosen Bicep and their benefits.

Bicep:

Bicep is a domain-specific language (DSL) that uses declarative syntax to deploy Azure resources. It provides concise syntax, reliable type safety, and support for code reuse. We believe Bicep offers the best authoring experience for your infrastructure-as-code solutions in Azure.

Also, you can say, Bicep is the next version of ARM(Azure Resource Manager).

Benefits of Bicep

  1. Support for all resource types and API versions.
  2. Simple syntax.
  3. Authoring experience.
  4. Modularity.
  5. Integration with Azure services.
  6. No state or state files to manage.
  7. No cost and open source.

If you want to know more about it’s benefits, checkout this link.

Before proceeding further, I’ll recommend you to go through my previous article to get better understanding on this article.

Pre-Requisite

  1. First, complete the all step process from my previous article.
  2. Get some basic knowledge on Bicep, ARM and pipeline YAML file.
  3. An Azure DevOps account.
  4. Installed Azure CLI on your system
  5. Any code editor. (Preferable: VSCode)

Well, if you are done with the above pre-requirement process, then,

Let’s get started

So, the below steps we’re going to cover in this article.

  • Conversion of ARM to Bicep template
  • Create YAML file for build pipeline
  • Run pipeline in Azure DevOps

Conversion of ARM to Bicep Template

Well, before converting ARM to Bicep template, we need to complete following steps. Let’s look into those.

Export ARM Template

Assuming, you have completed all steps from my previous article.

  • Navigate to Azure portal,
  • then, go to the Resource Group(RG) where your Grafana resources are present,
  • After this, click on the Export template under Automation section of Grafana RG,
  • then, click on Download and you will get your ARM template in zip folder,
  • at last, you have to extract the downloaded zip folder and you can see two files:
  1. parameters.json: Rather than passing parameters as inline values in your script, you can use a JSON file that contains the parameter values.
  2. template.json: It’s nothing but a ARM template which are JSON files that define the resources you need to deploy for your solution.

Note: Save these files for later use as we’re gonna work with these files.

Installation of Bicep

Installing bicep is pretty simple. Actually, there is a two ways to install bicep:

  1. Importing Bicep extension in VS code.

To import Bicep in VS code, do the following steps:

  • Open VS code,
  • Navigate to Extension, and search bicep,
  • Click on install button, and restart the VS code.

2. Run the az command.

az bicep install

To check the bicep version, run this command in terminal.

az bicep version

Convert ARM to Bicep template file

Let’s convert the downloaded ARM template to Bicep template file.
To convert it, you just need to run an az command,

az bicep decompile --file <arm_template_file>

Note: If you want , you can create bicep file from scratch. I didn’t do it as it’s a lengthy process.

With this, you can see the newly created .bicep file in your working directory.

To view the converted bicep file, checkout this link.

Now, will create a parameters.json file for passing our values to bicep file based on different environment.

To see the format of this file, you can checkout this link.

Create YAML file for Build Pipeline

Create Build Template file

Note: The template file may vary based on your requirement.

In this file, we’re gonna do the following things:

  1. Copy files, where bicep and parameters files are present,
  2. then, publish build artifacts into a folder.

So, here is the code(.json):

jobs:   - job: Build     displayName: Build     pool:      vmImage: $(vmImageName)     steps:   - task: CopyFiles@2     inputs:      SourceFolder: "grafana_resource_deploy"      Contents: "**"      TargetFolder: $(Build.ArtifactStagingDirectory)   - task: PublishBuildArtifacts@1     inputs:      PathtoPublish: $(Build.ArtifactStagingDirectory)      ArtifactName: "drop"      publishLocation: "Container"

Now, let’s create a deployment template file.

Create Deployment Template File

So, here, we’re gonna deploy Grafana resource into Azure portal using az command.

Here is the code(.json):

parameters:  - name: environment    type: string  - name: azureSubscription    type: string  - name: resourceGroup    type: stringjobs:  - job: grafana_infra    displayName: grafana_infra_release    pool:      vmImage: $(vmImageName)    steps:    - task: AzureCLI@2      inputs:       azureSubscription: ${{ parameters.azureSubscription }}       scriptType: 'bash'       scriptLocation: 'inlineScript'       inlineScript: |        az deployment group create --resource-group ${{     parameters.resourceGroup }} -f 
<bicep-file-location> --parameters <parameters-file-location>-${{ parameters.environment }}.json

Note: Here, Grafana resources are deploy on Resource Group level, but if you want you can do this on Subscription level also.

Create Build Pipeline file

This is the main pipeline file where i’ve created different stages which will trigger both build and deployment template to run in different enviroment.

So, here is the code(.yml):

trigger:  batch: true  branches:    include:    - main    - develop    - feature/*variables:  vmImageName: 'ubuntu-latest'stages:- stage: CopyingAndPublishingArtifact  displayName: Build and Publish  jobs:  - template: templates/build_template.yml- stage: DeployToDev  displayName: Deploy To Dev  dependsOn: CopyingAndPublishingArtifact  jobs:- template: templates/deployment_template.yml  parameters:     environment: dev     azureSubscription: $(azureSubscription-dev)     resourceGroup: $(resourceGroup-dev)- stage: DeployToTest  displayName: Deploy To Test  dependsOn: DeployToDev  jobs:  - template: templates/deployment_template.yml    parameters:      environment: test      azureSubscription: $(azureSubscription-test)      resourceGroup: $(resourceGroup-test)- stage: DeployToProd  displayName: Deploy To Prod  dependsOn: DeployToTest  jobs:- template: templates/deployment_template.yml  parameters:     environment: prod     azureSubscription: $(azureSubscription-prod)     resourceGroup: $(resourceGroup-prod)

Note: The variables which is used in above code are defined in the DevOps pipeline section, which will see it later.

Run pipeline in Azure DevOps

Note: Assuming that you have pipeline created in DevOps and deployment files are present in Repos.

Define Variables in Pipeline

As you can see, in the above code(.yml), I’ve used variables to access the value from pipeline at run time.

So, to add variable, do the following steps:

  • Navigate to the DevOps → Pipelines,
  • Select the required pipeline and click it,
  • Click on Edit button,
  • And, you can see the pipeline yml content which we’ve configured earlier. Then, click on Variables button.
  • Al last, add the variables and their respected values and Save it.

Run the Pipeline

After doing the above configuration, just do the following steps.

  • After defining variables, then click on Run button,
  • then, select the branch and click on Run button,
  • at last, you can see your running pipeline and it’s result.

Now, let’s verify that our Grafana resources are deployed successfully or not.
So, just jump to the Azure portal and navigate to the Resources Group which we have defined earlier in Variables section.

And, you can see something like this:

Note: You have to upload Grafana config and database file manually into the File Share under Storage account.

Conclusion

Well, that’s it!

You just deployed the Grafana resources through Bicep.
